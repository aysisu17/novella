# Руководство по стилю

Этот документ определяет стандарты кодирования для проекта "Novella". Соблюдение этих правил обеспечивает чистоту, читаемость и единообразие кодовой базы.

## 1. Язык

* **Комментарии к коду:** Все комментарии должны быть написаны **на русском языке**, чтобы соответствовать существующему коду и тексту игры.
* **Игровой текст:** Весь текст, выводимый пользователю (`Console.WriteLine`), должен быть на русском языке.

## 2. Именование

Мы следуем стандартным соглашениям C# (.NET):

* **Классы и Методы:** `PascalCase` (например, `Program`, `Main`).
* **Локальные переменные:** `camelCase` (например, `choice`, `hasOrb`).
* **Константы:** `PascalCase`.

### Именование меток (Labels)

Для сохранения структуры проекта, метки именуются следующим образом:

* **Шаги сюжета:** `PascalCase`, начиная со `Step` (например, `Step1`, `Step2`, `Step10`).
* **Концовки:** `PascalCase`, начиная с `Ending_` (например, `Ending_Victory`, `Ending_Void`).

## 3. Управление потоком (goto)

**Этот проект использует `goto` для управления повествованием.**

* **При добавлении нового контента:** Пожалуйста, следуйте существующему шаблону, используя `goto` для перехода между шагами.
* **Рефакторинг:** Если вы планируете заменить `goto` на более современную структуру (см. `ARCHITECTURE.md`), пожалуйста, сначала обсудите это в Issue, так как это крупное изменение.

## 4. Форматирование

* **Отступы:** Используйте 4 пробела для отступа (не табы).
* **Фигурные скобки:** Ставьте фигурные скобки на новой строке (стиль Allman), как в существующем коде `Program.cs`.

    ```csharp
    // ПРАВИЛЬНО
    if (choice == "1")
    {
        goto Step15;
    }
    else
    {
        goto Ending_Trapped;
    }

    // НЕПРАВИЛЬНО (Стиль K&R)
    if (choice == "1") {
        goto Step15;
    } else {
        goto Ending_Trapped;
    }
    ```
* **`Thread.Sleep`**: Используйте `Thread.Sleep` экономно, только для создания драматической паузы (как в `Step3:`). Не злоупотребляйте ими, чтобы не замедлять игру.
